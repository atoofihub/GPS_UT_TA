Collins Tropospheric Delay Example

Collins Model Description

The Collins (1999) tropospheric model is widely used in Satellite-Based Augmentation Systems (SBAS) such as WAAS and EGNOS. It estimates the total tropospheric delay (dry + wet) for GNSS signals using meteorological parameters interpolated from a global data table (Trop_Data.mat). The model uses a common mapping function for both dry and wet components, and accounts for seasonal and latitudinal variations in atmospheric conditions.

Data source: Meteorological parameters are interpolated from Trop_Data.mat based on the receiver's latitude. This file contains average and seasonal variation values for pressure, temperature, water vapor pressure, and lapse rates.

Step 1: Interpolate Meteorological Parameters

$$ \begin{align*} P_0, \Delta P, T_0, \Delta T, e_0, \Delta e, \beta_0, \Delta\beta, \lambda_0, \Delta\lambda = \text{interpolate from Trop\_Data.mat at } \varphi \end{align*} $$

The parameters are linearly interpolated from the data file for the given latitude. For this example:

$P_0 = 1016.15$, $\Delta P = $ (see data), $T_0 = 293.97$, $\Delta T = $ (see data), $e_0 = 21.21$, $\Delta e = $ (see data), $\beta_0 = 0.00594$, $\Delta\beta = $ (see data), $\lambda_0 = 3.1465$, $\Delta\lambda = $ (see data)

Step 2: Compute Seasonal Meteorological Values

$$ \xi(\varphi, D) = \xi_0(\varphi) + \Delta\xi(\varphi) \cos\left(\frac{2\pi(D - D_{min})}{365.25}\right) $$

Where $D_{min} = 28$ for northern latitudes. For this example:

$P = 1016.15$, $T = 293.97$, $e = 21.21$, $\beta = 0.00594$, $\lambda = 3.1465$

Step 3: Zero-altitude Vertical Delays

$$ \begin{align*} T_{rz0,d} &= 10^{-6} k_1 R_d \frac{P}{g_m} \\ T_{rz0,w} &= \frac{10^{-6} k_2 R_d}{(\lambda + 1) g_m - \beta R_d} \frac{e}{T} \end{align*} $$

Where $k_1 = 77.604$ K/mbar, $k_2 = 382000$ K$^2$/mbar, $R_d = 287.054$ J/(kg K), $g_m = 9.784$ m/s$^2$.

$T_{rz0,d} = 2.3136$ m, $T_{rz0,w} = 0.2036$ m

Step 4: Vertical Delays at Receiver Height

$$ \begin{align*} T_{rz,d} &= \left(1 - \frac{\beta H}{T}\right)^{\frac{g}{R_d \beta}} T_{rz0,d} \\ T_{rz,w} &= \left(1 - \frac{\beta H}{T}\right)^{\frac{g(\lambda+1)}{R_d \beta} - 1} T_{rz0,w} \end{align*} $$

Where $g = 9.80665$ m/s$^2$, $H$ is the receiver height in meters.

$T_{rz,d} = 2.1824$ m, $T_{rz,w} = 0.1615$ m

Step 5: Mapping Function (Obliquity Factor)

$$ M(E) = \frac{1.001}{\sqrt{0.002001 + \sin^2(E)}} $$

Where $E$ is the satellite elevation angle in degrees. For this example:

$M(E) = 1.9940$

Step 6: Total Tropospheric Delay

$$ T_r(E) = (T_{rz,d} + T_{rz,w}) M(E) $$

The total slant tropospheric delay for the given elevation angle:

$T_r(E) = 4.6736$ meters

Summary

  • The Collins model provides a reliable estimate of tropospheric delay for GNSS signals, using interpolated meteorological data and a common mapping function.
  • In this example, the total tropospheric delay is 4.67 meters for a receiver at 35° latitude, 500 m height, on day 100, with a satellite elevation of 30°.
  • Meteorological parameters are sourced from Trop_Data.mat, which should be updated for best accuracy in different regions.
